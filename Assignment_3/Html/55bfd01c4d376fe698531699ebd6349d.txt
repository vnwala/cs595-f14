		<p>I&#8217;ve been grabbing a bit of software from Technet (you know, that thing that Microsoft are shutting down! <_<) and with their download links, they provide SHA1 hashes of the ISOs. I had a quick look around the web for something that allowed you to get SHA1 hashes of files and while I found a few, I didn't find any that would allow you to provide a hash and compare the resultant hash with the one you're given by the provider so I decided to write one myself and as with any opportunity, I decided to use PowerShell and .NET with Windows Forms to create a GUI interface as opposed to it all being text based. Sacrilege to those PowerShell purists but it's a limited feature tool and I wanted to learn something new so here's the code.</p>
<pre class="brush: powershell; title: ; notranslate" title="">

		<h1 class="entry-title"><a href="http://www.lewisroberts.com/2014/04/02/html5-canvas-scaling/" rel="bookmark">HTML5 Canvas Scaling</a></h1>
		<div class="entry-meta">
			<span class="entry-date"><a href="http://www.lewisroberts.com/2014/04/02/html5-canvas-scaling/" rel="bookmark"><time class="entry-date" datetime="2014-04-02T23:56:01+00:00">Wed 2nd April, 2014</time></a></span> <span class="byline"><span class="author vcard"><a class="url fn n" href="http://www.lewisroberts.com/author/ljr/" rel="author">Lewis</a></span></span>			<span class="comments-link"><a href="http://www.lewisroberts.com/2014/04/02/html5-canvas-scaling/#respond" title="Comment on HTML5 Canvas Scaling">Leave a comment</a></span>
					</div><!-- .entry-meta -->
	</header><!-- .entry-header -->

		<div class="entry-content">
		<p>Recently I&#8217;ve had some time to fettle with things other than Infrastructure and Windows servers so I&#8217;ve decided to do <a href="http://www.eastgateit.co.uk/" target="_blank">my company website</a> with HTML 5. It&#8217;s nothing special but as part of my tinkering, I created a new logo and I thought it&#8217;d be cool to re-create it in HTML5 Canvas so it was unique and somewhat versatile.</p>
<p>The process starts out as quite a difficult task. How on earth do you re-create a logo by drawing some lines? Well, to cut a long story short, I opened it up in my favourite vector drawing app (the one I used to create it) and just copied the x,y coordinates of the anchor points on to an HTML5 canvas that was the same size as the artboard the illustration was on.</p>
<p>First you&#8217;ll need a canvas element on your HTML5 page. Something like:</p>
<p>Pay attention to the width and height declarations since this is the canvas size (in pixels) you&#8217;re drawing on.</p>
<pre class="brush: xml; title: ; notranslate" title="">
&lt;canvas id=&quot;logo&quot; height=&quot;102&quot; width=&quot;102&quot;&gt;Browser says no...&amp;lt;canvas&amp;gt;&lt;/canvas&gt;
</pre>
<p>The end result is pretty cool. All of this:</p>
<pre class="brush: jscript; title: ; notranslate" title="">

function drawLogo(target) {
var element = document.getElementById(target);
var canvas = element.getContext(&quot;2d&quot;);

/*
Originally drawn on 102px x 102px canvas (it wouldn't go smaller without distorting!)
Use this scale (the original canvas size)
to calculate the appropriate scaling values for
the intended target canvas.
*/

var oScaleY = 102;
var oScaleX = 102;

canvas.scale ((element.width / oScaleX), (element.height / oScaleY)); //Scale the canvas to fit the canvas object's width height.

canvas.beginPath();

canvas.moveTo(5, 4); // Starting Point
canvas.lineTo(5, 30);
canvas.lineTo(23, 30);
canvas.lineTo(23, 40);
canvas.lineTo(5, 40);
canvas.lineTo(5, 66);
canvas.lineTo(23, 66);
canvas.lineTo(23, 77);
canvas.lineTo(5, 77);
canvas.lineTo(5, 103);
canvas.lineTo(90, 103);
canvas.lineTo(98, 90);
canvas.lineTo(90, 77);
canvas.lineTo(79, 77);
canvas.lineTo(79, 66);
canvas.lineTo(84, 66);
canvas.lineTo(92, 53);
canvas.lineTo(84, 40);
canvas.lineTo(79, 40);
canvas.lineTo(79, 30);
canvas.lineTo(92, 30);
canvas.lineTo(100, 17);
canvas.lineTo(92, 4);

canvas.closePath();

// Seemingly the gradient size must be based on the original drawn canvas.
var grad = canvas.createLinearGradient(0,oScaleX,0,0);

grad.addColorStop(0, &quot;rgb(46,92,139)&quot;);
grad.addColorStop(0.25, &quot;rgb(0,113,188)&quot;);

canvas.fillStyle = grad; // Set the gradient as the fill colour.

canvas.fill(); // Fill the closed path (with the gradient)

canvas.fillStyle = &quot;rgb(255,255,255)&quot;; // Set a new fill colour (white)
canvas.fillRect(29, 30, 44, 10); // Create a filled (white) rectangle.

canvas.beginPath();

canvas.moveTo(29, 66);
canvas.lineTo(29, 77);
canvas.lineTo(64, 77);
canvas.lineTo(64, 81);
canvas.lineTo(73, 81);
canvas.lineTo(73, 62);
canvas.lineTo(64, 62);
canvas.lineTo(64, 66);

canvas.closePath();

canvas.fill();
}

drawLogo(&quot;logo&quot;); // Actually draw the logo

</pre>
<p>Gives you something looking like this:</p>
<p><img alt="" src="http://www.lewisroberts.com/wp-content/uploads/2014/04/040214_2354_HTML5Canvas1.png" /></p>
<p>Anyway, I&#8217;m not here to teach you how to re-create my company logo in HTML5, needless to say, that&#8217;s what it looks like. The JavaScript function above does some clever scaling so if your intended target HTML5 canvas element defines its height and width as something other than what you drew the original anchor points on, it&#8217;ll scale to that appropriately. To explain, I originally drew this on a 102px x 102px canvas so all the canvas.lineTo declarations relate to that size canvas. Say I want to scale it up to say 300px x 300px, I can use the canvas.scale method to achieve that so this bit of code calculates the scaling factor and applies that to the canvas which helps to retain quality when it&#8217;s scaled unlike bitmaps which, as we know, don&#8217;t scale well.</p>
<pre class="brush: jscript; title: ; notranslate" title="">
/*
Originally drawn on 102px x 102px canvas (it wouldn't go smaller without distorting!)
Use this scale (the original canvas size)
to calculate the appropriate scaling values for
the intended target canvas.
*/

var oScaleY = 102;
var oScaleX = 102;

canvas.scale ((element.width / oScaleX), (element.height / oScaleY)); //Scale the canvas to fit the canvas object's width height.
</pre>
<p>Now, as we look at the site on a desktop PC, the logo looks great and is razor sharp. It&#8217;ll scale up nicely in multiples of 2 by adjusting the canvas element&#8217;s width and height properties and retain its sharpness but values other than that will give you blurred edges. Still, SVG does the same thing as far as I can tell from my own eyesight.</p>
<p>So what&#8217;s the problem? Well, given that the new site is supposed to be responsive design and degrade (I think I&#8217;ll call it re-grading since there&#8217;s no loss of functionality for my site) for mobile and tablet, when you look at that image on a mobile or tablet, it looks blurred because the devicePixelRatio is greater than 1 and the image is scaled up by the device but retains the original dimensions as defined on the canvas element itself.</p>
<p>On a Nexus 5 with a devicePixelRatio of 3 (at least that&#8217;s what its browser tells me it is) this blurriness is quite profound and looks something like this which, for a sharp site, looks shocking compared to the HTML5 SVG above.</p>
<p><img src="http://www.lewisroberts.com/wp-content/uploads/2014/04/nexus5_screenshot.jpg" alt="nexus5_screenshot" width="382" height="396" class="alignnone size-full wp-image-798" /></p>
<p><img src="http://www.lewisroberts.com/wp-content/uploads/2014/04/nexus7_screenshot.jpg" alt="nexus7_screenshot" width="254" height="282" class="alignnone size-full wp-image-799" /></p>
<p>So, the fix to me seems logical: you should scale the canvas element up by the declared devicePixelRatio amount, let the scaling calculation in the drawing function handle the actual size that comes out and scale back using CSS.</p>
<p>So, here&#8217;s the extra code before we call our drawLogo function:</p>
<pre class="brush: jscript; title: ; notranslate" title="">
// Upscale the canvas element for mobile HiDPI screens.
// It gets scaled down again to the original size by CSS.
// Keeps the HTML5 canvas sharp on all screen types.

var canvas = document.getElementById(&quot;logo&quot;);
canvas.width = canvas.width * window.devicePixelRatio;
canvas.height = canvas.height * window.devicePixelRatio;
</pre>
<p>To retain the 102px x 102px image size that we originally intended for our canvas (remember this works for other sizes too!) and for it to look good (not just a bigger image) we need to use CSS to keep the canvas element&#8217;s dimensions at what we originally intended for our canvas.</p>
<pre class="brush: css; title: ; notranslate" title="">
canvas {
height: 102px;
width: 102px;
}
</pre>
<p>In essence what I&#8217;ve done is to increase the dimensions of the canvas element proportionally to the devicePixelRatio so, if the devicePixelRatio is 3, the in-flight JavaScript resizes the entire element to: width = (102 x 3), height = (102 x 3). The increased size of the canvas element is passed to the drawLogo function which creates an image 3 times the size. This is then drawn on to the canvas which is then styled using CSS back down to 102px. The best way to think of this is like using an oversized PNG scaled down within the img dimensions so it looks OK on mobile devices.<br />
The end result is images that are sharp and consistent across desktop, tablet and mobile screens:</p>
<p><img src="http://www.lewisroberts.com/wp-content/uploads/2014/04/nexus7_improved.jpg" alt="nexus7_improved" width="347" height="333" class="alignnone size-full wp-image-800" /></p>
<p><img src="http://www.lewisroberts.com/wp-content/uploads/2014/04/nexus5_improved.jpg" alt="nexus5_improved" width="400" height="431" class="alignnone size-full wp-image-801" /></p>
<p><img src="http://www.lewisroberts.com/wp-content/uploads/2014/04/desktop_original.jpg" alt="desktop_original" width="163" height="163" class="alignnone size-full wp-image-802" /></p>
<p>NB: Please remember that I&#8217;ve only been able to test this with the limited hardware available to me, all of which are Android devices. Nexus 5, Nexus 7 (2012 and 2013), Samsung Galaxy Nexus and of course my Windows 8.1 desktop. If you get different or inconsistent results with your devices, please let me know!</p>

<article id="post-695" class="post-695 post type-post status-publish format-standard hentry category-general">
	
	<header class="entry-header">

		<h1 class="entry-title"><a href="http://www.lewisroberts.com/2013/04/18/active-directory-recycle-bin/" rel="bookmark">Active Directory Recycle Bin</a></h1>
		<div class="entry-meta">
			<span class="entry-date"><a href="http://www.lewisroberts.com/2013/04/18/active-directory-recycle-bin/" rel="bookmark"><time class="entry-date" datetime="2013-04-18T14:45:54+00:00">Thu 18th April, 2013</time></a></span> <span class="byline"><span class="author vcard"><a class="url fn n" href="http://www.lewisroberts.com/author/ljr/" rel="author">Lewis</a></span></span>			<span class="comments-link"><a href="http://www.lewisroberts.com/2013/04/18/active-directory-recycle-bin/#respond" title="Comment on Active Directory Recycle Bin">Leave a comment</a></span>
					</div><!-- .entry-meta -->
	</header><!-- .entry-header -->

		<div class="entry-content">
		<p>Windows Server 2008 R2 delivered a new feature called the Active Directory Recycle Bin which offers the ability to restore items deleted from the Active Directory database by restoring them from the Recycle Bin with the simplicity of&#8230;.well, it&#8217;s not really that simple.</p>
<p>The premise is simple enough. You&#8217;ve deleted an item that you want to restore so instead of breaking out the backups, taking down a Domain Controller, booting in to DSRM and re-acquanting yourself with NTDSUTIL, you enable the Recycle Bin to save you all that hassle.</p>
<p>But wait a minute! Before enabling the Active Directory Recycle Bin (ADRB) there are a couple of caveats which you should be aware of. Now, Microsoft will tell you what you <em><strong>need</strong></em> to enable your use of ADRB such as:</p>
<ul>
<li>Forest Functional Level: Windows Server 2008 R2.</li>
<li>All Domain Controllers running Windows Server 2008R2</li>
</ul>
<p>&#8230;but the limits that enabling Active Directory Recycle Bin can have on restore operations is significant enough to ensure that your Backup Operators and Data Security personnel need to be consulted before you make a unilateral decision to enable it.</p>
<ol>
<li>Enabling ADRB transitions all currently <em>Tombstoned</em> (deleted) objects to the new <em>Recycled</em> object state. This effectively means that current Tombstoned objects (objects deleted in the last 180 days) should never be restored, either through object reanimation or via an authoritative restore.</li>
<li>Similar to the above, once an object reaches the <em>Recycled</em> object state (after 180 days of being a <em>Logically Deleted</em> object) it cannot be restored or recovered from backup. Microsoft recommends that you do not use authoritative restores at all after enabling ADRB and that you only use ADRB to restore objects during their deleted object lifetime (DOL). This article: http://technet.microsoft.com/en-us/library/dd379542(WS.10).aspx details the recommendation which effectively means that restores must be done within the deleted object lifetime or you should consider the object completely unrecoverable. The deleted object lifetime can be adjusted at the expense of an increased AD database size and replication traffic but the default is 180 days.</li>
<li>ADRB cannot restore changed objects &#8211; this must be done using an authoritative restore while the object is still live. Hopefully the proper use of change processes in your organisation should  minimise the eventuality of this occurring and permit the ability to simply undo a change but we all know what happens in the real world.</li>
<li>Enabling ADRB results in the size of your Active Directory database increasing (and consequently the replication bandwidth requirements) to accommodate the new object states before objects deleted are completely removed from the database. The increase is dependent on the amount and type of objects created and deleted but since there is a new object state, the time the objects remain in the database is effectively doubled.</li>
</ol>
<p>Once each of these discussion points has been thoroughly considered should you look at enabling the Active Directory Recycle Bin.</p>
<p>I know this subject is fairly old hat given that Windows Server 2012 is now available but I&#8217;m still astonished by the numbers of Active Directories that I come across that aren&#8217;t making use of the Active Directory Recycle Bin. Reading the pro-tip (can I call myself a pro?) enabling it in Windows Server 2012 is pretty much a no-brainer with the easy-peasy GUI on offer, just be mindful of the implications.</p>
<p><strong>Pro tip</strong>: Although the procedure for using the Recycle Bin is currently based on PowerShell, Windows Server 2012 provides a Graphical User Interface to permit much simpler use of the Recycle Bin feature.</p>
<p><strong>Admission</strong>: I actually wrote this article nearly 12 months ago but never finished it or published it &#8211; since then Windows Server 2012 has been released so I&#8217;ve made mention of that in the article.</p>

		<h1 class="entry-title"><a href="http://www.lewisroberts.com/2012/09/21/fedora-16-to-17-upgrade-shenanigans/" rel="bookmark">Fedora 16 to 17 upgrade shenanigans</a></h1>
		<div class="entry-meta">
			<span class="entry-date"><a href="http://www.lewisroberts.com/2012/09/21/fedora-16-to-17-upgrade-shenanigans/" rel="bookmark"><time class="entry-date" datetime="2012-09-21T23:03:57+00:00">Fri 21st September, 2012</time></a></span> <span class="byline"><span class="author vcard"><a class="url fn n" href="http://www.lewisroberts.com/author/ljr/" rel="author">Lewis</a></span></span>			<span class="comments-link"><a href="http://www.lewisroberts.com/2012/09/21/fedora-16-to-17-upgrade-shenanigans/#comments" title="Comment on Fedora 16 to 17 upgrade shenanigans">1 Comment</a></span>
					</div><!-- .entry-meta -->
	</header><!-- .entry-header -->

		<div class="entry-content">
		<p>Fedora 16 to 17 Upgrade&#8230;nightmare.</p>
<p>I&#8217;m convinced that persons developing and using Linux on a daily basis insist that upgrading your Linux distro to the latest and greatest is supposed to be the easiest thing since wiping your own arse. In all frank and honesty, upgrading to Fedora 17 from 16 has been a royal pain in said arse.</p>
<p>I first attempted to perform the upgrade shortly after Fedora 17 went live. I received some random error message about gtk_init moaning about displays or some such bollocks and didn&#8217;t bother trying any further thinking they&#8217;d sort something out with F16 to help you through the upgrade. Well, they didn&#8217;t.</p>
<p>Running preupgrade on a runlevel 3 (non-GUI, you know, a server) will give you said error. What you&#8217;re supposed to do is execute preupgrade-cli. Why of course! I should have known! Did the dev forget to check the runlevel and offer a bit of advice instead of puking all over the screen? Guess not.</p>
<p>Then when you run preupgrade-cli, you do so by specifying the distro to upgrade to:</p>
<pre class="brush: plain; title: ; notranslate" title="">preupgrade-cli &quot;Fedora 17 (Beefy Miracle)&quot;</pre>
<p>Then you&#8217;re moaned at for not having a large enough partition for some random file but told it will be all OK if you&#8217;re on a wired connection. Full speed ahead your system downloads all the packages and then, after saying its made a change to your grub config, you are safe to reboot. Now the system reboots&#8230;and promptly goes straight back in to F16. If your computer could look back at you in a nonplussed fashion, it would be doing.</p>
<p>So you have another go and spot the error that says sh grub file or folder not found. Eh? How is my PC even booting if grub isn&#8217;t there? It turns out that F16 apparently forced an upgrade to GRUB2 and it&#8217;s your GRUB2 config file that receives the updated &#8220;Upgrade to Beefy Miracle&#8221; boot entry. Useful! Except my computer was booting from GRUB. Don&#8217;t ask me why, I have no idea.</p>
<p>So I ensure GRUB2 is installed and on my boot partition:
<pre class="brush: plain; title: ; notranslate" title="">grub2-install /dev/sda</pre>
<p>I then decide it&#8217;s wise to ensure that my GRUB2 install has the latest info about my system (all the kernels I had in GRUB):
<pre class="brush: plain; title: ; notranslate" title="">grub2-mkconfig -o /boot/grub2/grub.cfg</pre>
<p>A reboot yields an actual option to perform the upgrade to Beefy Miracle so I select it and again my computer screen is puked on with various 404 errors. Yes, that&#8217;s right, the &#8220;second stage&#8221; URL that grubby spewed in to my grub.cfg at the end of preupgrade-cli was WRONG so I had to edit it and remove the /LiveOS\squashfs.img location MANUALLY by pressing &#8220;e&#8221; on the highlighted boot table entry when GRUB2 loaded.</p>
<p>Only after all this (which I admit involved a lot of reading/learning) did I manage to get to a point where it looked like Fedora 17 would install the packages it had spent an hour downloading. As I type this it&#8217;s still installing the F17 packages but I have no idea if it will actually work given how many upgrade cock-ups I&#8217;ve had to battle my way through so far. Someone at Fedora needs shooting.</p>
<p>&#8211; Lewis</p>
